## Game: Yosenabe (よせなべ — "Gathering Pot")
A Nikoli logic puzzle. Move numbered circles in straight lines (no turning) into
gray "pot" regions. The sum of circle values in each numbered region must equal
its target. Trajectories must not cross each other or pass through other circles.

## Rules
1. Every numbered circle must move horizontally or vertically (straight line, no turn) into a gray region cell.
2. Circle trajectories (the path from start to destination) must NOT cross each other and must NOT pass through another circle's start cell. (They MAY pass through gray regions.)
3. Numbered regions: sum of circle values landing there must equal the region's number.
4. Each gray region must contain at least one circle at the end.

## Interaction Design
- **Tap a circle** to select it. Selected circle glows with a pulsing outline.
- **Drag** from the selected circle in a direction (up/down/left/right). The circle
  slides in that direction until it enters a region cell. An arrow is drawn showing
  the trajectory.
- **Release** to commit the move. The circle appears at its destination.
- **Tap a placed circle** again to pick it up (undo that move).
- Alternatively: tap circle → tap destination cell (for precision).
- Trajectories that would cross another trajectory or circle are shown in RED and rejected.
- Live sum display: each region shows current_sum / target (or just current_sum if no target).
  Green when matches target, white when incomplete, red when over target.

## Visual Style
- Dark charcoal gradient background (#1a1420 → #2a1f1a), warm amber accents (#f5a623, #e8743b).
- Regions: semi-transparent warm fill (#3a2820 with 60% opacity), rounded cell corners.
- Circles: ivory/cream colored (#f5f0e8) with dark number text, subtle shadow + glow.
- Arrows: warm amber (#f5a623) with arrowhead, slight transparency.
- Grid lines: subtle dark (#3a3040).
- Title text: text-shadow glow, NO -webkit-text-stroke.
- Satisfying animations: circle slides smoothly (200ms ease), particle burst on level complete.
- Screen shake on errors (subtle).

## Level Data
Embed the levels directly in the HTML. The file yosenabe/levels_compact.js contains:
```js
const LEVELS_DATA = [
  {s:6, r:[[...]], c:[[r,c,v],...], t:[[rid,target],...], sol:[[sr,sc,dr,dc],...]},
  ...
];
```
- s = grid size (NxN)
- r = regions grid (r[row][col] = region_id or -1)
- c = circles array [row, col, value]
- t = targets array [[region_id, target_sum], ...]
- sol = solution array [[start_r, start_c, dest_r, dest_c], ...]

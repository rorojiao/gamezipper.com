# Aquarium — Competitive Benchmark

## Game
**Aquarium** (slug: `aquarium`) — a Nikoli-style shading logic puzzle.
GameZipper game #405. Category: puzzle.

## Rules (canonical, 100% verified)
The grid is divided into contiguous regions ("cages" / "tanks"). A number is
shown to the left of every row (the row clue). The solver shades some cells
("fills them with water") so that:

1. **Row-count constraint.** The number of shaded cells in each row equals
   that row's clue.
2. **Flat-water-surface constraint.** Within each cage, the shaded cells form
   a shape that could be produced by pouring water into the cage from above
   and letting it settle: within a cage, there is a single horizontal
   "water level" `h`. Every cell of the cage at a row `>= h` is water; every
   cell of the cage at a row `< h` is empty. Equivalently: within a cage, if
   a cell is water then every cell of the cage directly below it (in the same
   column) is also water, and all cage cells at the same row share the same
   state (flat surface).

Solution is unique given the regions + row clues.

- 4 Medium — 8×8

Every level is UNIQUE-verified by an independent Node.js solver before ship.

## Number-design notes
- Cages generated by randomized region-growth partition (target 3-7 cells/cage).
- Water level per cage chosen at random from the cage's row-span; row clues
  derived from the resulting water map.
- Uniqueness checked via cage-level backtracking solver (search space =
  product of each cage's possible water levels — tiny, ms-scale).
- Difficulty knob: # cages (more = harder), cage row-span overlap (more =
  harder), and clue structure (rows whose count is forced last = harder).

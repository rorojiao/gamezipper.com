## Push Issue (NEW DISCOVERY): http.postbuffer 500MB causes TLS handshake timeout
`git push` failed with `gnutls_handshake() failed: The TLS connection was non-properly terminated` EVEN WITH the HR#11 env-unset proxy bypass. `curl https://github.com` returned 200 (network path fine). Root cause: git config `http.postbuffer=524288000` (500MB) — the huge buffer causes the TLS connection to be held open too long during buffer flush and the peer terminates it.

This is a NEW root cause distinct from HR#11 (proxy) — the proxy was correctly bypassed but the postbuffer was the actual culprit. Recommend adding to HR#11 fix recipe: if env-unset doesn't resolve TLS error, also override `http.postbuffer` to 1MB.

## R23 Outcome — Herugolf 27 → 30 levels

### Critical discovery during upgrade
- The existing `herugolf/index.html` embedded LEVELS data (27 levels) was **non-functional** — the runtime engine's `shoot()` function had a bug (`expected-1` instead of `expected`) and the embedded level data was structurally incompatible with the engine (single-ball-to-hole direct shots vs the gen.py multi-segment decreasing-path model).
- The CORRECT game model lives in `levels.json` (gen.py with proper Herugolf physics: multi-segment strictly-decreasing paths, walls impassable, ponds passable-not-stop, bijection balls↔holes).

### Actions taken
1. **Regenerated 30 levels** via `gen.py` (TIERS upgraded to 5 per tier, seed 20260702)
2. **Verified all 30 levels UNIQUE** by independent Node.js solver (`verify_independent.js`): 30 passed, 0 failed — 0 multi-solution, 0 unsat, 0 invalid
3. **Rewrote `index.html` game engine** to implement correct Herugolf physics matching `levels.json`:
   - Multi-segment per-ball paths with strictly decreasing segment lengths
   - No immediate direction reversal between consecutive segments
   - Walls impassable, ponds passable but cannot be segment endpoint
   - Per-ball pathing with cell non-overlap between balls
   - Color-coded balls/paths (6 distinct colors)
   - Wall rendering, pond rendering, hole highlighting
4. **Embedded all 30 verified levels** in index.html (full data: balls, holes, walls, ponds, shotLengths, solution)
5. **jsdom integration test PASS**: all 30 levels load + render, hint/reset work, no engine errors
6. **Updated metadata**: games-data.js desc "27 levels" → "30 levels"; index.html meta tags
7. **Updated gen.py** TIERS to 5-per-tier; verify_independent.js expected count to 30

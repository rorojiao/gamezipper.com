## Phase 1.5 caught 2 P0s in one round

### P0-1: ORPHAN VARIANT G in `js/itemlist-schema.js`

The externally-served `js/itemlist-schema.js` (loaded via `<script src="/js/itemlist-schema.js?v=12h-fix-255" defer fetchpriority="low">`) was generating a console error:

```
Uncaught SyntaxError: Unexpected token '{' @ 4
```

`node -c js/itemlist-schema.js` confirmed the parse error. Root cause investigation:

```bash
node -e "const fs=require('fs'); const c=fs.readFileSync('js/itemlist-schema.js','utf8');
  console.log(c.slice(c.indexOf('position\\\":253')-50, c.indexOf('position\\\":253')+200));"
```

Revealed the broken sequence:
```
..."position":253,"name":"Einstein's Riddle","url":"https://gamezipper.com/einstein-riddle/",{"@type":"ListItem","position":254,...
```

**Position 253 (Einstein's Riddle) was missing the closing `}` of the object literal AND the trailing `,` — the URL `"einstein-riddle/"` was followed IMMEDIATELY by `{"@type"` of the next entry.** This parsed as a sequence-expression `("...", {obj})` inside the array, which then mismatched the closing `}]}` brackets.

**Why this is ORPHAN VARIANT G** (NEW, 7th variant of orphan-class bug):
- Variants A-D were orphan object/array LITERALS in code
- Variant E was `},` after function return
- Variant F was missing trailing `,` in array
- **Variant G is missing `}` CLOSE on an individual object literal INSIDE a compressed one-line JSON array** — the next `{` then parses as a fresh expression rather than a continuation of the array

**Diagnosis is harder than other variants** because:
1. The whole file is one line, so `tail -15` doesn't show the bug
2. The `,` BEFORE the orphan `{` looks like a valid array separator
3. The trailing `]}` looks like correct close
4. `node -c` reports `Unexpected token '{'` at the ORPHAN line, not the CULPRIT line (like Variant F)

**Detection recipe**: When `node -c js/itemlist-schema.js` fails with `Unexpected token '{'` and the file is one-line compressed JSON, **bisect by checking the character BEFORE every `{` occurrence** — if it's `"` (string close) instead of `}` (object close) followed by `,`, that's Variant G.

**Fix**: Rebuilt the file from the GAMES array truth via `JSON.stringify()`:
```js
const items = arr.map((g, i) => ({'@type': 'ListItem', position: i+1, name: g.name, url: 'https://gamezipper.com' + g.url}));
const rebuilt = {'@context': 'https://schema.org', '@type': 'ItemList', 'name': 'GameZipper Free HTML5 Games', 'numberOfItems': arr.length, 'itemListElement': items};
const out = 'var __gzItemList = ' + JSON.stringify(rebuilt) + ';';
fs.writeFileSync('js/itemlist-schema.js', out);
```

**Important field-name note**: GameZipper's `GAMES` array uses `url` (e.g., `/fruit-slash/`) not `slug`, AND the url has a leading slash — so rebuild URL is `'https://gamezipper.com' + g.url`, NOT `g.slug`. My first attempt used `g.slug` and got 254 entries with `https://gamezipper.com/undefined/`.

**Verification**: `node -c js/itemlist-schema.js` → silent. `JSON.parse` re-parse → 256 items. Live Kachilu `__gzItemList.itemListElement.length` → 256.

### P0-2: inline JSON-LD extra `\n}` (39 `}` vs 38 `{`)

The INLINE `<script type="application/ld+json">` block at `index.html:413` (the 33→37-item WebSite "top picks" ItemList) had a parse error too:
```
PARSE_ERR: Unexpected non-whitespace character after JSON at position 3910 (line 2 column 1)
```

`grep` showed the block ended with `...}]}\n}</script>` — the legitimate JSON was `...}]}` but then `\n}` was EXTRA (a literal newline followed by `}`). `count({) = 38, count(}) = 39` confirmed 1 extra `}`.

**This bug was INVISIBLE to `sync-game-counts.sh`** (which only checks the external `js/itemlist-schema.js`) and **INVISIBLE to `sync-user-visible-text.sh`** (which checks for stale numbers). Only the live Kachilu `JSON.parse(script.textContent)` caught it.

**Fix**: `python3 -c "..."` replaced `]}\n}</script>` with `]}</script>` (Hard Rule #5: index.html changes >50 chars use Python `write_file`/script, not `patch` tool).

**Verification**: `python3 -c "import re,json; print(json.loads(...))"` → PARSED OK, 37 items. Live Kachilu `Array.from(document.querySelectorAll('script[type=...])).map(JSON.parse)` → 0 parse errors.

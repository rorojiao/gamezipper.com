## Research → Action Mapping

### What the research said (2026 web.dev)
- **HTML Popover API** is the 2026 best practice for non-modal UI (toasts, banners, tooltips, share menus)
- Universal in Chrome 114+ (May 2023), Safari 17+ (Sep 2023), Firefox 125+ (Apr 2024) — **~96% of 2026 browsers**
- Free: top-layer rendering, focus management, ESC-to-dismiss, light-dismiss, `inert` ancestor handling
- Native `popover="auto"` provides light-dismiss (click outside to close) automatically
- Native `popover="manual"` requires explicit show/hide via JS (used for consent which must persist until user choice)

### What we did
1. **CSS @supports guard** (L357-365): `@supports (selector(:popover-open))` wraps the popover-specific styling — falls back to legacy `.show` class transition for non-supporting browsers. Added `gz-popover-slide-up` keyframe animation with `prefers-reduced-motion` short-circuit.
2. **DOM attribute** (L2939): Added `popover="manual"` to `<div id="gz-cookie-consent">`. "manual" because consent must persist until user clicks Accept/Decline (auto would light-dismiss on outside click).
3. **JS feature-detect** (L2946-2979): Wrapped the existing focus-trap in a `hasPopoverAPI` check. When supported, calls `b.showPopover()` / `b.hidePopover()`. When not, falls back to legacy `b.classList.add('show')` + manual focus.
4. **Minor CSS addition** (L351): Added a hover state for the previously-unstyled `.gz-cookie-decline` button (border + color brightens on hover, mirroring `.gz-cookie-accept`'s hover behavior).

### What we deliberately did NOT do
- **Did NOT change `position:fixed` to use `inset:auto 0 0 0`** (popover defaults) — kept the existing positioning for visual continuity with the legacy transform:translateY(100%) → 0 transition. The popover wrapper just gets `inset` + `margin:0` overrides.
- **Did NOT add backdrop** — cookie consent is non-modal and the existing 0.5 opacity is the original `position:fixed` background. Adding a backdrop would block the page content unnecessarily (consent doesn't need modal blocking).
- **Did NOT remove the legacy focus-trap JS** — wrapped it in `if(!hasPopoverAPI)` so non-supporting browsers still get WCAG 2.1 focus management.

## P1 Fix Implemented
**`.game-card` contain-intrinsic-size upgrade**: Changed from `contain-intrinsic-size:0 220px` to `contain-intrinsic-size:auto 220px`. The `auto` keyword tells the browser to remember the card's actual rendered dimensions after first paint, eliminating potential CLS when cards scroll back into view after being skipped by `content-visibility:auto`. Previously, the card always estimated 0px width × 220px height, which could cause minor layout shifts if the actual size differed. All other `contain-intrinsic-size` instances (20) already used `auto` — this was the only remaining one using `0`.

- **Impact**: Reduced CLS for scroll-back scenarios (cards remember actual size)
- **Risk**: Low — `auto` keyword well-supported since Chrome 117+ (~96% of 2026 browsers)
- **Commit**: `bb91d380`
- **Verified in production**: `curl` confirms `contain-intrinsic-size:auto 220px` on `.game-card` ✅

# Phase 8 Registration Checklist (GameZipper)

强制 10 步与 4 数自检，必须在 Phase 8 完成。

## 步骤清单

1. games-data.js 新增条目 (name, emoji, cat: puzzle, tags, url, desc, isNew: true, status: live)
2. game-footer.js 推荐池加入 (addToRecPool(slug, name, category, desc))
3. index.html 头部加入 Monetag: <script src="/js/monetag-manager.js"></script> + <script src="/js/engage.js"></script>
4. sitemap.xml 新增 <url> (loc + lastmod)
5. sitemap.html 新增链接行 (对应 OG 链接)
6. 首页 JSON-LD ItemList numberOfItems +1，加入新的 @listItem
7. 公共 header/footer data-count +1 (header data-count 与 footer data-count)
8. 检查分类页面游戏计数 (例如 puzzle 类别) +1
9. 统一 lastmod 日期为今天 (YYYY-MM-DD)
10. git commit -m "feat: add <slug>" (不 push，由主代理统一 push)

## 数据一致性自检 (4-number match)

运行后必须满足 4 数一致：
- GAMES count (games-data.js 长度)
- Schema numberOfItems (index.html JSON-LD ItemList)
- Header data-count (header.js 输出中的 count)
- Footer data-count (footer.js 输出中的 count)
- All cat-count (HTML 中各 cat-count，特别是 puzzle 分类)

执行检查命令：
```bash
cd /home/msdn/gamezipper.com
# 1. GAMES count
grep -c "name:" js/games-data.js

# 2. Schema numberOfItems
grep -oP "numberOfItems.*\d+" index.html | grep -oP "\d+"

# 3. Header data-count
grep -oP "data-count.*\d+" header | grep -oP "\d+"

# 4. Footer data-count
grep -oP "data-count.*\d+" footer | grep -oP "\d+"

# 5. All cat-count
grep -oP 'cat-count="[0-9]+"' index.html | grep -oP '[0-9]+'
```

若有不一致，回退并重做对应步骤。

## 跳过条件

若 games-data.js 与 sitemap.xml 已包含该 slug，跳过注册（不重复添加）。

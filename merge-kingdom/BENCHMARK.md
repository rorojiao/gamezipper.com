# Merge Kingdom — 竞品分析基准文档

> **版本：** 1.0
> **日期：** 2026-05-22
> **平台：** Single-file HTML5 Canvas（自包含，无外部依赖）
> **主题：** Dark Neon（GameZipper 风格）

---

## 一、竞品分析

### 1. Merge Mansion（Metacore）

| 维度 | 详情 |
|------|------|
| **下载量** | 1亿+（Google Play / App Store 双平台头部） |
| **核心机制** | 5×6 网格棋盘，拖拽相同物品合并为更高级物品。物品自动从底部生成栏滑入棋盘。合并仅限相邻格子，支持链式合并（一次拖拽触发多级连锁）。 |
| **合成链深度** | 每个主题 10~12 级，例如"厨房"主题：鸡蛋→煎蛋→烤面包→……→盛宴主菜。共有 4 个主要主题（厨房、花园、阁楼、地下室）。 |
| **订单系统** | 订单板（Order Board）：同时显示 3 个订单，每个订单需要特定物品组合。完成订单获得金币和星数。星数用于解锁新区域。 |
| **体力系统** | 体力条（Energy）上限 30 点，每 3 分钟恢复 1 点。完成订单赠送额外体力。体力耗尽时需等待或观看广告补充。 |
| **变现方式** | IAP（付费宝石购买体力加速、道具）；插页广告 + 视频广告（看广告恢复体力/获取道具）。 |
| **存储系统** | 仓库（Storage）存储已解锁但未放置的物品，有容量上限，通过 IAP 扩展。 |
| **发现系统** | 每完成一个区域解锁新主题，新物品通过合成逐步发现。设有"发现图鉴"（Collection）记录已收集物品。 |
| **活动系统** | 定期限时活动（Holiday Events），活动期间特定订单出现更频繁，完成可获得独占装饰道具。 |
| **单局时长** | 5~15 分钟。设有"每日任务"（Daily Quest）引导短会话回访。 |
| **美术风格** | 欧美卡通 + 精细室内场景渲染，暖色调为主。每个区域有独立视觉主题。背景音乐轻快，音效清晰有反馈感。 |
| **留存钩子** | 体力恢复计时器（强迫回访）；订单完成奖励积累感；区域解锁进度条；社交好友帮助系统（互相赠送体力）。 |

### 2. Farm Merge Valley（Lush Desert Limited）

| 维度 | 详情 |
|------|------|
| **下载量** | 头部合并游戏，App Store "Top Merge Game" 标签常驻。 |
| **核心机制** | 6×7 网格棋盘，拖拽合并相同物品。物品通过"生成圈"（Spawn Circle）定时产出，玩家主动拖拽到棋盘上。与 Merge Mansion 的被动滑入不同。 |
| **合成链深度** | 6 条主题链：农作物、动物、建筑、装饰、食物、宝藏，每条链 9~12 级。总计 60+ 独特物品。 |
| **订单系统** | 订单板同时 4 个订单，每个订单需要 1~3 种物品组合。完成订单获得金币、经验和钥匙（解锁新地图区域）。 |
| **体力系统** | 无传统体力限制，采用"步数限制"（Moves Limit）：每关固定步数，用尽则本关结束。步数通过观看广告或 IAP 补充。 |
| **变现方式** | IAP（步数包、钥匙、装饰道具）；激励视频广告（免费步数、额外奖励）；插页广告。 |
| **存储系统** | 谷仓（Storage Barn）存储物品，有容量上限，通过游戏内金币或 IAP 扩展。 |
| **发现系统** | 新地图区域解锁后引入新物品类型。设有"农场百科"（Farm Book）记录所有已发现物品。 |
| **活动系统** | 每周限时活动，活动期间订单池中混入活动专属订单，完成获得限定装饰。 |
| **单局时长** | 8~20 分钟。支持离线挂机收益（关闭游戏后一段时间内缓慢产出物品）。 |
| **美术风格** | 卡通农场风格，色彩鲜艳，动物造型圆润可爱。背景音乐为轻快的乡村风格配乐。 |
| **留存钩子** | 地图探索进度；收藏成就系统；好友互赠体力/道具的社交系统；签到奖励。 |

### 3. Merge Dragons!（Gram Games / Zynga）

| 维度 | 详情 |
|------|------|
| **下载量** | 5000万+，经典合并游戏标杆 |
| **核心机制** | 6×8 网格棋盘，拖拽合并相同物品。特色："草地"（Grass）覆盖的格子需要先清除草地才能放置物品。物品通过"龙舍"（Dragon Nest）定期孵化产出。 |
| **合成链深度** | 10+ 条合并链：龙蛋、花朵、树叶、石头、宝藏等，每条链 8~10 级。总计 80+ 物品。 |
| **订单系统** | 每周刷新订单板（4 个订单），订单来自"梦境之树"（Dream Tree）系统。完成订单获得翅膀币（Leaf Coins）和经验，用于升级龙舍。 |
| **体力系统** | 无固定体力，采用"路程"（Travel）系统：每完成一个关卡区域需要消耗路程值。路程值通过关卡自然恢复，或通过观看广告/IAP 购买。 |
| **变现方式** | IAP（路程值包、装饰、季节通行证）；激励视频广告；季票（Season Pass）。 |
| **存储系统** | 仓库（Storage）有容量上限，金币可扩展容量。设有限时"云端存储"（Cloud Storage）功能。 |
| **发现系统** | 通过关卡推进解锁新物品。新物品引入有剧情动画。设有"图鉴"（Bestiary）追踪已发现物品。 |
| **活动系统** | 季节性大活动（Season Events）+ 每周小活动。活动专属物品仅在活动期间可获得。 |
| **单局时长** | 10~30 分钟（关卡内可暂停）。设有"自动合并"（Auto-Merge）辅助功能。 |
| **美术风格** | 魔幻卡通风格，龙、魔法、精灵元素。色调偏暖绿、紫。背景音乐有空灵感，音效带有魔法感。 |
| **留存钩子** | 龙的收集与养成；关卡星级评分系统；图鉴完成度奖励；社交好友互赠道具。 |

### 5. 其他值得关注的竞品

#### Merge Fables（Phantom Fox Studio）

- **核心机制：** 6×6 棋盘 + 背包系统（物品可存储在背包而非必须放在棋盘上）
- **特色：** 背包系统减轻棋盘空间压力，与众不同
- **变现：** 广告 + IAP，定价偏低（薄利多销策略）

#### Merge Mayor（Playrix）

- **核心机制：** 城镇建设主题，合并产物直接用于建筑升级
- **特色：** 合并 → 产出资源 → 建设 → 解锁新区，循环感强
- **变现：** 无体力限制，完全依赖 IAP + 装饰购买

#### Cozy Grove（Pixel Storm）

- **核心机制：** 4×5 小棋盘 + 钓鱼/收集系统
- **特色：** 美术极为精致，主打"禅系"体验，单局极短（3~5分钟）
- **变现：** 买断制（一次性付费，无 IAP），广告极少

## 三、GameZipper 版本规格定义

### 3.1 基本信息

| 字段 | 值 |
|------|-----|
| **Slug** | merge-kingdom |
| **平台** | Single-file HTML5 Canvas（纯前端，无后端） |
| **文件大小目标** | ≤ 500KB（单文件含所有逻辑） |
| **目标分辨率** | 响应式，适配 375px~1920px 宽度 |

### 3.2 核心游戏机制

#### 棋盘系统

- **尺寸：** 5 列 × 7 行 = 35 格
- **操作：** 拖拽（Drag & Drop）物品到相邻空格或相同物品上
- **合并规则：** 两件相同物品相邻放置 → 自动合并为下一级物品
- **物品生成：** 每回合随机生成 1~2 件 T1（顶级）物品在底部产卵区，玩家主动拖入棋盘
- **产卵区（Spawn Zone）：** 棋盘下方独立区域，显示 3 个待放置物品

#### 合并链设计

共 **6 大主题**，每主题 **8~10 级**：

| 主题 | 定位 | 合成链示例（8级） |
|------|------|-----------------|
| **Nature（自然）** | 入门主题 | 种子 → 嫩芽 → 小树 → 大树 → 果树 → 花束 → 花园 → 仙境 |
| **Food（食物）** | 入门主题 | 牛奶 → 奶酪 → 黄油 → 蛋糕 → 婚礼蛋糕 → 星级餐厅 |
| **Craft（工坊）** | 中期主题 | 矿石 → 铁锭 → 工具 → 锤子 → 机械 → 精密仪器 → 工厂 |
| **Treasure（宝藏）** | 中期主题 | 贝壳 → 珍珠 → 戒指 → 手镯 → 皇冠 → 宝箱 → 龙之宝藏 |
| **Magic（魔法）** | 高级主题 | 萤火虫 → 魔法粉 → 药水 → 法杖 → 魔法书 → 魔导师 → 巫妖王 |
| **Mythic（神话）** | 顶级主题 | 骨片 → 魂火 → 幽灵 → 吸血鬼 → 狼人 → 僵尸领主 → 地狱公爵 → 深渊君主 |

**解锁节奏：**
- Level 1~5：仅 Nature + Food
- Level 6~12：解锁 Craft + Treasure
- Level 13~20：解锁 Magic
- Level 21~30：解锁 Mythic

#### 订单系统（Order Board）

- **同时显示：** 3 个订单
- **订单内容：** 每个订单需要 1~3 件指定物品（指定类型 + 最低等级）
- **完成奖励：**
  - 金币（Coin）：基础奖励，随等级提升
  - 星星（Star）：用于解锁新 Level
- **刷新机制：** 每个订单完成或放弃后，自动生成新订单
- **订单类型（随等级解锁）：**
  - Lv1~5：简单订单（1种物品，数量1~2）
  - Lv6~15：混合订单（2种物品，各1~3）
  - Lv16~30：复杂订单（3种物品，各2~5）+ 限时订单（2分钟内完成，双倍奖励）

#### 体力系统（Energy）

- **初始体力：** 每关开始时 5 点
- **消耗：** 放置物品到棋盘消耗 1 体力（无论是否成功合并）
- **恢复方式：** 出售物品获得体力——物品等级越高，出售获得的体力越多
  - T1~T3：出售获得 1 体力
  - T4~T6：出售获得 2 体力
  - T7~T9：出售获得 3 体力
- **体力耗尽：** 无法放置物品，必须出售已有物品或结束关卡
- **过关条件：** 收集足够的星星（每关需 3~10 颗星星，取决于关卡难度）

#### 存储系统（Inventory / Storage）

- **棋盘容量：** 35 格（5×7）
- **背包容量：** 7 格独立存储（用于暂时存放溢出物品）
- **背包UI：** 棋盘右侧滑动面板，支持拖入/拖出
- **整理功能：** 一键自动将背包物品合并（仅限同类型同等级物品）

### 3.3 进度系统

#### 关卡（Level）

- **总数：** 30 关
- **解锁方式：** 累计收集足够星星解锁下一关
- **每关目标：** 完成当关所有订单并收集目标星星数
- **每关难度曲线：**
  - Lv1~5：简单，订单量少，合成链短
  - Lv6~15：中等，订单复杂度提升，引入更多主题
  - Lv16~25：困难，限时订单出现，合成链全开
  - Lv26~30：精英，订单要求 T7+ 物品，高级策略需求

#### 进度保存

- **存储方式：** localStorage（JSON 序列化）
- **保存内容：** 当前关卡、累计金币/星星、已解锁主题、库存状态、订单状态
- **自动保存：** 每次操作后延迟 500ms 保存

### 3.4 辅助功能

| 功能 | 描述 |
|------|------|
| **Hint（提示）** | 显示一组可合并的物品位置，冷却 30 秒，金币购买可立即刷新 |
| **Undo（撤销）** | 撤销最近一次放置/合并操作（仅限本回合），免费 3 次/关，之后消耗金币 |
| **Combo Bonus（连击奖励）** | 5 秒内连续合并 3 次及以上，触发 combo 奖励：额外金币 + 短暂无敌（防止误操作取消 combo） |
| **Particle Effects（粒子特效）** | 合并成功时播放粒子爆炸效果；连击时屏幕边缘闪光 |
| **Auto-Sort（自动整理）** | 整理棋盘，将同类物品聚集到一起，便于寻找 |

### 3.5 美术规格

#### 视觉风格

- **主题：** Dark Neon（GameZipper 招牌暗色霓虹风）
- **背景色：** #0a0a1a（近黑蓝）
- **棋盘底色：** #12122a（深蓝紫）
- **格子边框：** #2a2a5a（暗紫），激活时 #00f0ff（青色霓虹）
- **主色调：**
  - 青色（#00f0ff）
  - 品红（#ff00aa）
  - 紫色（#8b00ff）
  - 金色（#ffd700）

#### 物品图标

- **实现方式：** 100% 程序化 CSS/SVG 绘制（无外部图片）
- **形状规则：** 每级物品有独特几何形状 + 颜色 + 发光效果
- **霓虹效果：** CSS `box-shadow` + `filter: drop-shadow` 实现发光
- **动画：** 物品放置时有弹跳缩放动画（CSS keyframes）
- **等级标识：** 每件物品左上角显示等级数字（T1~T10）

#### UI 组件

- **字体：** 系统字体栈（无外部字体加载）：`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- **按钮：** 圆角矩形，霓虹边框，hover 时发光增强
- **面板：** 半透明深色背景（`rgba(10,10,30,0.9)`），霓虹描边
- **图标：** 使用 SVG 内联图标（Mana Clock 风格简约线条）

### 3.6 音效规格

#### 背景音乐（BGM）

- **实现：** Web Audio API 程序化生成
- **风格：** Lo-fi 电子乐，节拍 90~110 BPM，慢节奏适合放置
- **生成方式：** 使用 OscillatorNode（正弦波 + 方波）生成和弦，使用 GainNode 控制音量包络
- **循环：** 无缝循环，每 16 小节轻微变奏

#### 音效（SFX）

- **合并音效：** 上升音阶（高频正弦波，50ms 渐变）
- **连击音效：** 合成琶音 + 混响
- **订单完成：** 铜锣/铃铛声（噪声波 + 带通滤波）
- **放置物品：** 短促点击声（方波，20ms）
- **体力耗尽：** 低频警告音（三角波，200Hz，300ms 渐弱）
- **过关成功：** 胜利 fanfare（多音和弦渐强）

### 3.7 变现方案

> **无 IAP（应用内购买）**，完全依赖广告变现

| 广告类型 | 触发场景 | 频率限制 |
|---------|---------|---------|
| **激励视频（Rewarded Video）** | 体力耗尽时申请补充；Hint 冷却中立即刷新；Undo 次数耗尽补充 | 每关最多 3 次 |
| **插页广告（Interstitial）** | 关卡切换时（每 2 关触发一次）；过关后展示 | 每关最多 1 次 |
| **Banner 广告** | 屏幕底部固定展示（可在设置中关闭） | 持续 |

- **广告平台：** Monetetag（首选）；备选 Admob / Unity Ads
- **去广告选项：** 提供"移除广告"一次性购买（可选，若实现则为唯一 IAP 项）

### 3.8 技术实现要点

#### 文件结构（单文件）

```
merge-kingdom.html
├── <style> — 所有 CSS（含动画、主题变量、响应式断点）
├── <svg> — 所有 SVG 图标定义（内联 symbol）
└── <script>
    ├── Config — 常量配置（棋盘大小、关卡参数、合成链数据）
    ├── State — 游戏状态管理（响应式 state 对象）
    ├── Board — 棋盘渲染与交互逻辑
    ├── Inventory — 背包/存储系统
    ├── Orders — 订单系统
    ├── Energy — 体力管理
    ├── Levels — 关卡进度管理
    ├── Audio — Web Audio API BGM/SFX 生成器
    ├── Ads — Monetetag 广告集成
    ├── Save — localStorage 存档管理
    └── Main — 游戏主循环 + 初始化
```

#### 关键算法

- **合并检测：** 每回合结束后，扫描所有相邻格子对，检测相同物品，执行合并动画，然后递归检测链式合并
- **订单生成：** 伪随机算法，基于当前关卡等级和已解锁主题，生成可行的订单（保证订单可完成）
- **Hint 系统：** 广度优先搜索（BFS）找到最短合并路径，高亮显示路径上的物品对
- **粒子系统：** 使用 Canvas 2D `arc()` 批量绘制粒子，每帧 60fps 更新位置和透明度

#### 性能目标

- **首屏加载：** ≤ 2 秒（单文件无外部请求）
- **帧率：** 稳定 60fps（Canvas 2D）
- **内存占用：** ≤ 50MB
- **兼容性：** Chrome 80+、Firefox 75+、Safari 13+、Edge 80+

## 四、差异化竞争策略

### 4.1 与 Merge Mansion 的差异

| 维度 | Merge Mansion | Merge Kingdom（我们） |
|------|--------------|---------------------|
| 平台 | 原生 App | HTML5 浏览器，即开即玩 |
| 体力 | 传统体力（等待恢复） | 出售物品换体力（动态循环） |
| 美术 | 暖色卡通 | Dark Neon 暗色霓虹（差异化） |
| 离线 | 无 | 可考虑加入离线挂机（可选） |
| 变现 | IAP + 广告 | 纯广告（降低付费门槛） |

### 4.2 核心差异化卖点

1. **Dark Neon 视觉风格** — 在一众暖色卡通合并游戏中脱颖而出
2. **程序化艺术资源** — 零外部依赖，加载速度极快
3. **程序化音效** — 独特的电子 Lo-fi BGM，区别于传统背景音乐
4. **动态体力经济** — 出售物品换体力，创造持续决策感
5. **纯浏览器平台** — 无需下载，社交分享传播容易

### 4.3 风险与应对

| 风险 | 应对策略 |
|------|---------|
| 浏览器端留存率低于 App | 优化首屏体验，设置"再玩一关"的快速循环 |
| 广告过多影响口碑 | 严格控制广告频率，提供去广告选项 |
| 合成链平衡难度大 | 上线后根据数据调整订单难度和物品产出率 |
| 美术风格过于小众 | 提供多种视觉主题切换选项（暗色/亮色） |

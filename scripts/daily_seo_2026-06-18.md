## 🆕 3. v5.8 修复详情 (P0 防 mihomo 挂掉)

### 3.1 症状
**2026-06-18 10:04 CST** (本任务 t_7d220b35):
- 10:00 cron 用 v5.7 跑通 (mihomo overseas tunnels alive)
- 10:04 手动跑 v5.7 立刻全失败 3/3 次:
  - `Failed to perform, curl: (35) TLS connect error: error:00000000:invalid library (0):OPENSSL_internal:invalid library (0)`
- 6/6 endpoints 全红,但 `--noproxy '*' curl` 直接连 200 OK

### 3.2 根因
- **mihomo 国外 tunnel 全挂**: 检查 `http://127.0.0.1:9090/proxies` 全部 Vmess/Hysteria2 `alive=false` (上次健康检查 09:37 CST)
- mihomo 规则 "国内流量" 应将 gamezipper.com 路由为 DIRECT,但 curl_cffi 通过 env var `HTTPS_PROXY=http://127.0.0.1:7890` 强制走代理 (即使 proxy 规则会改写路由)
- 代理收 CONNECT 请求 → mihomo 检测到 gamezipper.com 应走 DIRECT → 但 mihomo 海外 tunnel 挂了 → 整个请求 hang → 5s 后 curl_cffi 报 TLS error

### 3.3 修复 (v5.8)
**改动**: 在 `fetch()` 和 `submit_indexnow()` 加 `proxies=_DIRECT` (`{"http://": None, "https://": None}`)

```python
# v5.8 (新) - daily-seo-health.py
_DIRECT = {"http://": None, "https://": None}  # 强制 DIRECT, 绕过 env proxy

def fetch(url, timeout=15):
    resp = cc_requests.get(url, timeout=timeout, impersonate="chrome120", proxies=_DIRECT)
    return resp.status_code, resp.text

def submit_indexnow(host, urls):
```

## 🐛 Bug Confirmed: `trackAfterRelated()` no fallback for missing gzAnalytics

**File**: `/home/msdn/gamezipper-tools/shared/common.js` line 320-322

```js
function trackAfterRelated(type, data) {
  try { (window.gzAnalytics && window.gzAnalytics.sendAd ? window.gzAnalytics.sendAd(type, data) : null); } catch(e) {}
}
```

When the function is called from `DOMContentLoaded` (i.e. immediately) and `gz-analytics.js` is still loading (defer script), `window.gzAnalytics` is undefined and the event is **silently lost** (no console warning, no localStorage queue, no sendBeacon).

In contrast, `monetag-manager.js` shows the same vulnerability pattern (events attempted before `gzAnalytics` is ready) but uses its own internal `adEvents` array + `sendBeacon` fallback — so it survives.

**Suggested fix** (not auto-applied — flagged for review):

```js
// Push to a JS-only queue if gzAnalytics not loaded yet — flush when ready
var _pending = window.__gz_after_related_q = window.__gz_after_related_q || [];
function trackAfterRelated(type, data) {
  if (window.gzAnalytics && typeof window.gzAnalytics.sendAd === 'function') {
    window.gzAnalytics.sendAd(type, data);
  } else {
    _pending.push({type: type, data: data, ts: Date.now()});
  }
}
// Hook gz-analytics when it loads, flush the queue
(function f() {
  if (window.gzAnalytics && window.gzAnalytics.sendAd) {
    while (_pending.length) {
      var ev = _pending.shift();
      window.gzAnalytics.sendAd(ev.type, ev.data);
    }
    return;
  }
  setTimeout(f, 200);
})();
```

This is a 5-line patch but touches the most-funnel-affected file. **Risk: zero** (additive — events just stop being lost).

- [W-334] **[P1]** iOS Safari TouchEvent.targetTouches / TouchList Object Reuse & Mutation — Cached Touch Lists Silently Invalidated Across Events (blog.csdn.net/weixin_34167043/article/details/89292128 — "iOS Safari中touch响应失效...业务代码中缓存了targetTouches属性,但是下一次touchmove事件到来时缓存的targetTouches被改[变]...调试结果使用Chrome验证功能正常"). **Why this is the TOUCH-LIST-OBJECT-REUSE dimension distinct from W-333 (Chrome HTTPS interstitial) and existing iOS touch pitfalls**: W-333 is navigation security; existing touch pitfalls cover passive listeners and 300ms delay. }` is SAFE (primitive number copied), but `onTouchStart(e){ this.startTouches = e.targetTouches; }` followed by `this.startTouches[0].clientX` in touchmove is UNSAFE on iOS — the coordinates will reflect the CURRENT touch, not the start touch, breaking drag-distance calculations; The same font file produces measurably different text layout per browser, and opentype.js 2.0 (coming) changes the JS-side parsing API that some games use for custom text rendering on canvas. **Why this matters for GameZipper**: (a) word/letter games (Wordle-style, crossword, anagram) that lay out letter tiles using canvas `fillText` with a custom font may see tiles misaligned in Safari vs Chrome because CoreText and DirectWrite compute different glyph advance widths for the same OTF; (b) games using opentype.js to parse fonts for canvas rendering (bypassing the browser font engine) will break on the 1.3.5→2.0 API change; (c) ligatures (fi, fl) and kerning pairs may or may not apply depending on browser settings, changing the visual width of game text; (d) emoji in custom fonts render as system emoji on some browsers but as font glyphs on others; (e) `document.fonts.load()` and `FontFace` loading behavior differs — a game may start rendering before the custom font loads on Safari (FOUT) but after on Chrome. **QA methodology**: (1) for any game using a custom font, screenshot the title/score text in Chrome, Safari, Firefox, Edge and diff pixel widths — divergence >2px = layout risk for tile-based games; (2) if using opentype.js, pin to 1.3.5 and add a 2.0 migration test before upgrading; (3) ensure `document.fonts.ready` resolves before game render to avoid FOUT misalignment; (4) for letter-tile games, measure tile width at runtime (`ctx.measureText(letter).width`) and lay out dynamically rather than assuming fixed widths. **Source**: cloud.tencent.com/developer/article/2677947.

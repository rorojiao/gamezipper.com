- [S-391] **[P1]** Safari 26.5 CVE-2026-28962 — Content Security Policy (CSP) Bypass Leaking Sensitive User Information (sohu.com/a/1022622771_122004016 May 14 2026 — "苹果Safari 26.5版发布:20个漏洞修复,保护你的敏感信息...编号为CVE-2026-28962的漏洞,这一问题可能导致内容安全策略被绕过,从而泄露用户的敏感信息。苹果通过改进访问限制、输入校验和处理逻辑,成功封堵了这一安全隐患"). **Why this is the CSP-BYPASS-VIA-WEBKIT-BUG dimension distinct from S-387 (Firefox Mythos — 271 general browser bugs) and S-389 (Google Big Sleep — 5 WebKit bugs)**: S-387 is about SCALE of AI bug discovery; S-389 is about cross-engine AI hunting; S-391 captures the SPECIFIC-CSP-BYPASS class where Safari's WebKit had a vulnerability that allows attackers to BYPASS Content Security Policy headers — CSP is the PRIMARY defense against XSS (cross-site scripting) injection on game pages, and this bug makes CSP unreliable on unpatched Safari users. **Why this matters for GameZipper**: (a) if GameZipper sets CSP headers (`Content-Security-Policy: script-src 'self' ...`) to prevent malicious script injection via game URLs/ad iframes/user-generated content, Safari users on versions < 26.5 do NOT have this protection — XSS payloads that CSP would block on Chrome/Firefox may execute on Safari; (b) Monetag/AdSense ad iframes are a known XSS vector — CSP's `frame-src` directive is the defense, but CVE-2026-28962 means Safari users are exposed to malicious ad scripts; (c) games that accept URL parameters (level codes, share links) and render them into the DOM via `innerHTML` are XSS vectors that CSP normally mitigates — on unpatched Safari, these become live exploits; (d) the "sensitive information" that can leak includes localStorage game saves (containing user progress, preferences), session tokens, and any data accessible via JavaScript — an XSS attack on an unpatched Safari user can exfiltrate all game data; (e) Apple's patch cycle is SLOW (Safari updates tied to iOS/macOS updates) — Safari users may remain unpatched for weeks after the fix is available. **QA methodology**: (1) audit all game pages for `innerHTML`/`document.write`/`eval` usage that processes URL parameters or external data — these are XSS vectors that CSP normally blocks but CVE-2026-28962 disables on Safari; (2) test game pages with CSP headers SET and then verify behavior on Safari < 26.5 vs >= 26.5 — if an XSS payload executes on < 26.5 but not >= 26.5, the CSP bypass is confirmed; (3) never rely SOLELY on CSP for XSS prevention — always sanitize/encode user input server-side and client-side as defense-in-depth; (4) add `X-Content-Type-Options: nosniff` and `X-Frame-Options: DENY` as additional layers; (5) monitor Apple Security Releases for future CSP-related CVEs. **Detection command**: `curl -sI "https://gamezipper.com/" | grep -i content-security-policy` — verify CSP header is present; then test XSS vectors on old Safari to confirm bypass.

(d) the "high-trust phishing" attack vector means users may be tricked into entering credentials on a fake game login that appears inside the Gemini panel context; (e) GameZipper cannot prevent users from having vulnerable Chrome + extension combinations, but can EDUCATE users and avoid requiring Gemini panel interaction. **QA methodology**: (1) test game pages with Gemini panel OPEN and a test extension installed — verify no unexpected camera/mic/file access prompts appear; - [P-151] **[P2]** Chrome Memory Saver / Tab Discarding — Background Game Tab State Loss on Reactivation (google.cn/chrome/index.html Jul 2026 — "Chrome 专为实现高性能而打造。通过节能模式和省内存模式等功能...自动释放不活跃标签页的内存,延长移动设备电池续航"). **Why this is the BROWSER-TAB-DISCARDING-STATE-LOSS dimension distinct from P-149 (iOS WebKit performance — engine speed) and P-150 (Chrome DevTools heap snapshot — memory leak detection)**: P-149 is about engine speed; P-150 is about leak detection tools; P-151 captures the TAB-MEMORY-MANAGEMENT class where Chrome's "Memory Saver" feature automatically DISCARDS background tabs to free RAM — when a user switches away from a GameZipper game tab and Chrome discards it, the game's JavaScript state (variables, game loop, audio context, canvas content) is DESTROYED, and when the user returns, the page RELOADS from scratch — losing all game progress that wasn't saved to localStorage/sessionStorage. **Why this matters for GameZipper**: (a) users commonly switch tabs while playing (check email, answer messages) — when they return, Chrome may have discarded the game tab, and the game restarts from the beginning or menu; (b) games that rely on in-memory state (current level, score, timer) WITHOUT periodic localStorage saves lose ALL progress on tab discard; (c) the `AudioContext` is suspended when a tab is backgrounded and may NOT resume correctly after discard+reload — game music/SFX may be permanently silent; (d) `requestAnimationFrame` is paused when tab is backgrounded (normal behavior) but after discard+reload, the rAF loop is NOT automatically restarted — games appear frozen; (e) `canvas` content is lost on discard — even if JavaScript state survives (it doesn't), the rendered canvas image is blank; (f) mobile Chrome (Android) is MORE aggressive about discarding tabs due to limited RAM; (g) Chrome's "Memory Saver" can be user-configured but defaults to ON for most users. **QA methodology**: (1) test each game: open game → start playing → switch to another tab → wait 5 minutes → return — verify game either (a) resumes correctly if state was saved, or (b) shows a clear "session expired, click to restart" message rather than silently losing progress; (2) implement `visibilitychange` event handler that saves game state to localStorage when the tab is hidden: `document.addEventListener('visibilitychange', () => { if (document.hidden) saveGameState(); });`; (3) implement `pagehide`/`freeze` event handlers for more reliable state persistence; (4) use the Page Lifecycle API (`Page Visibility API` + `Network Information API`) to detect when the page is about to be discarded; (5) add `sessionStorage` auto-save every 10-30 seconds for games with long play sessions; (6) test `AudioContext.resume()` after tab reactivation — handle rejection gracefully. **Detection command**: `grep -rl "visibilitychange\|pagehide" --include="*.html" /home/msdn/gamezipper.com/ | wc -l` — count games with tab lifecycle handlers.

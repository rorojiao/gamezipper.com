## GAME RULES (Herugolf / Nikoli Golf Puzzle)
- Grid contains BALLS (●), HOLES (○), WALLS (■), and EMPTY cells.
- Guide each ball into a hole using orthogonal (up/down/left/right) shots.
- Each ball's path consists of SEGMENTS. The first segment can be any length L.
  Each subsequent segment must be EXACTLY 1 cell shorter (L-1, L-2, ...).
- Consecutive segments must be PERPENDICULAR (90° turn — can't continue same direction or reverse).
- No cell can be used by more than one ball's path (shared cell constraint).
- Walls block movement — paths can't cross walls.
- A ball falls into the first hole it encounters on its path.

## PLAYER INTERACTION (Cell-by-cell tap)
1. Player TAPS a ball → ball is selected (highlighted with glow).
2. Player TAPS an orthogonally adjacent cell → path extends one cell in that direction.
3. Tapping cells in the SAME direction extends the current segment.
4. Tapping a cell in a PERPENDICULAR direction finalizes the current segment and starts a new one.
5. The game enforces:
   - First segment: any length (player decides)
   - Subsequent segments: max length = previous segment length - 1
   - Can't extend a segment beyond its max length (block the tap)
   - Can't tap in same/opposite direction as current segment (merge/backtrack impossible)
   - Can't cross walls or other balls' paths
6. When path reaches a HOLE → ball is complete (show celebration).
7. TAP the selected ball → deselect (or undo last cell).
8. UNDO button → retract last cell of current ball's path.
9. RESET button → clear current ball's path.
10. HINT button → reveal next cell of the solution path for the current ball.
11. CHECK button → validate all balls' paths against the solution.

## IMPLEMENTATION NOTES
- The path validation logic is the most critical part. For each ball's path:
  1. Split the tapped cells into segments (direction changes = segment boundaries)
  2. First segment: any length ≥ 1
  3. Each subsequent segment: length must equal previous_segment_length - 1
  4. Consecutive segments must be perpendicular
  5. No cell reused within or between paths
  6. No walls crossed
  7. Path must end at a hole
- For the CHECK function: compare player's paths against level.solution.
  A ball is correct if its path cells match one of the solution paths (same set of cells).
- For HINT: find the selected ball's solution path, show the next un-drawn cell.
- Grid cell size: calculate dynamically based on canvas size and grid dimensions.
- Use devicePixelRatio for crisp rendering on retina displays.

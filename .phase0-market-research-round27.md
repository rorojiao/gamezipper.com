### TOP 5 GAP CANDIDATES — Scored on 25-Point Rubric

Scoring criteria (5 points each):
- **Uniqueness** (U): How absent from catalog
- **Trend/Searched** (T): Search volume and trend signal
- **Technical Fit** (TF): Feasibility as single-file HTML5
- **Monetization Potential** (MP): Ad placement, session length, replayability
- **Player Appeal** (PA): Casual appeal, learning curve, visual satisfaction

---

#### CANDIDATE 1: **Conveyor Belt Sort** (工厂传送带分拣)

**Concept**: Items travel on conveyor belts. Player rotates/swaps track switches to route items into the correct collection bins by color/shape/destination. Levels escalate: single belt → branching belts → speed-ups → multiple sorting criteria simultaneously.

**Gap Score**:
- U: 5/5 — completely absent, zero matches in catalog
- T: 4/5 — "conveyor sort"/"factory sort" is a trending hyper-casual mechanic (popular mobile apps)
- TF: 5/5 — Canvas/HTML5 rendering, physics = simple kinematics (no real physics engine needed)
- MP: 4/5 — Interstitial ad after every 5 levels, banner at bottom, high session length
- PA: 5/5 — Very satisfying, visually clear, easy to learn, hard to master

**Total: 23/25** ✅ EXCEEDS THRESHOLD

**Competitor analysis**: Apps like "Goods Sort", "Sort & Run" use this mechanic but as 3D mobile games. Web version with clean 2D top-down view would be unique in browser-game space.

**Why it fits**: Combines the popular "sort" mechanic (134 existing games prove demand) with a fresh delivery system (conveyor routing) that NO existing game uses. The routing/switching adds spatial reasoning depth beyond simple color-sort.

---

#### CANDIDATE 2: **Domino Toppler** (多米诺连锁)

**Concept**: Place dominoes on a grid, then "push" the first one to start a chain reaction. Goal: knock all target pieces or reach the goal tile. Levels include ramps, splitters (one domino hits two), bridges, and obstacles.

**Gap Score**:
- U: 5/5 — no physics domino chain game exists
- T: 3/5 — domino chain games have nostalgic appeal but not currently trending
- TF: 4/5 — Physics simulation needed, but can be simplified to 2D angular momentum
- MP: 4/5 — Level-based, good for interstitials
- PA: 4/5 — Satisfying chain reactions, but setup phase may bore some players

**Total: 20/25** ✅ EXCEEDS THRESHOLD

---

#### CANDIDATE 3: **Conveyor Factory Builder** (自动化工厂)

**Concept**: Place machines (cutter, painter, rotator, packer) along a conveyor belt. Raw materials enter → each machine transforms them → finished product exits. Like a 2D simplified Factorio puzzle.

**Gap Score**:
- U: 5/5 — completely absent
- T: 4/5 — automation/programming games trending (Factorio, Satisfactory, Mindustry influence)
- TF: 3/5 — Complex: multiple machine types, transformation logic, rendering pipeline
- MP: 3/5 — Niche audience, shorter sessions expected
- PA: 3/5 — Appeals to engineering-minded players, but steep learning curve

**Total: 18/25** ✅ MEETS THRESHOLD (barely)

---

#### CAND传送带 Sort dominates with 23/25, the clear winner.

### DECISION

**Selected Candidate: Conveyor Belt Sort** (slug: `conveyor-sort`)

Score: **23/25** — highest gap score achieved in any round.
- Uniqueness: 5/5 (zero matches in 525-game catalog)
- Trend: 4/5 (hyper-casual sort mechanic trending on mobile)
- Technical Fit: 5/5 (Canvas 2D, simple kinematics, no heavy physics engine)
- Monetization: 4/5 (level-based, interstitials, banner)
- Player Appeal: 5/5 (visually satisfying, easy to learn, hard to master)

**Why**: Combines proven demand (sort = 134 existing games, the #1 mechanic family in catalog) with a delivery system that's completely novel in the catalog (conveyor belt routing). The mechanic is trending on mobile but has zero web/browser presence. Single-file HTML5 implementation is straightforward.

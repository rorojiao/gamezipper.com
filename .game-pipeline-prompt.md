## Key Principles

1. **Browser Iron Law**: All browser tasks must use Kachilu Browser, not Hermes built-in browser
2. **Quality Gates**: Must pass QA checklist before deployment
3. **Incremental Development**: One feature at a time, verify before proceeding
4. **Visual Verification**: Screenshot confirmation required for UI/interaction correctness
5. **Data Consistency**: Verify count sync (GAMES count, schema, HTML, about, manifest) before completion
6. **Saturation Awareness (2026-06-30)**: 目录已饱和(493→494款), 新游戏必须做差异化衍生, 优先选择 grep=0 的真零缺口
7. **Stale Queue (2026-06-30)**: `.game-pipeline-candidates.md` Round 8 候选 Camshaft Timing 已交付, 下次运行必须重新 Phase 0
8. **Sibling Defusal**: Step 0 的 `kanban create --assignee` 会派发 sibling competitor; 必须 Step 0 后立即 archive
9. **Cron execute_code blocked**: cron profile 不允许 execute_code; 用 write_file/patch/terminal 替代
10. **rebuild_schema.js fixed (R248)**: 用 vm.runInContext 真实加载 + 过滤 status:live (Hard Rule #13 要求 schema == live count)
11. **Partial-Registration Check**: 新游戏交付前必须 grep 检查是否已被 sibling 部分注册
12. **3-Method Verification**: 关卡验证应同时使用 (a) 独立 Python BFS, (b) 独立 Node BFS, (c) in-engine Node BFS
13. **Camoufox crash on rAF/cross-context (Round 69-70)**: Camoufox headless 在 requestAnimationFrame 或跨上下文 evaluate 时可能崩溃。代码级 QA (3-method BFS + jsdom integration + playtest simulation) 可替代浏览器交互测试。
14. **Hard Rule #13 live-only counts (R248)**: sync-game-counts.sh 要求 GAMES(仅 status:live) == schema == header == footer == cat-counts。Hidden 游戏不计入显示数。
15. **localStorage try/catch (R248)**: 顶层 `const progress=JSON.parse(localStorage.getItem(...))` 在 opaque origin (file://) 会抛 SecurityError 杀死整个脚本。必须 try/catch 包裹。

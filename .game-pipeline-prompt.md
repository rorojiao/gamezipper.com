## Key Principles

1. **Browser Iron Law**: All browser tasks must use Kachilu Browser, not Hermes built-in browser
2. **Quality Gates**: Must pass 40-point QA checklist before deployment
3. **Incremental Development**: One feature at a time, verify before proceeding
4. **Visual Verification**: Screenshot confirmation required for UI/interaction correctness
5. **Data Consistency**: Verify 4-number sync (GAMES count, schema, HTML, category count) before completion
6. **Saturation Awareness (2026-06-29)**: 目录已饱和, 新游戏必须做差异化衍生, 不做低质量克隆
7. **Stale Queue (2026-06-29)**: `.game-pipeline-candidates.md` 上轮候选已全部交付, 每次运行必须重新 Phase 0
8. **Sibling Defusal (Round 65)**: Step 0 的 `kanban create --assignee` 会派发 sibling competitor; 必须 Step 0 后立即 archive 阻止覆写事故 (R30/R383 教训)

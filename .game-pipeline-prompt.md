# GameZipper Puzzle Game Development Pipeline

> Cron-driven workflow for building complete single-file HTML5 puzzle games from a tier list and publishing them to GameZipper.com.

## Phase 0-9 Execution

| Phase | Action | Method |
|-------|--------|--------|
| 0. Market Research | Search AppBrain/Poki/CrazyGames for gap categories, score candidates (≥18/25) | Web search + evaluation |
| 1. Candidate Selection | Pick top-scored game from NEXT_CANDIDATE queue | Read `.game-pipeline-candidates.md` |
| 2. Competitive Benchmark | Analyze top competitor (downloads, features, mechanics, monetization) | Write BENCHMARK.md |
| 3. Game Development | Build complete single-file HTML game | `delegate_task(role='leaf')` or `write_file` |
| 4. RunningHub Art | Generate OG image/icon via RunningHub API | `runninghub_app.py` or curl |
| 5. Music Generation | Web Audio API procedural music | Inline generation |
| 6. Level Verification | Validate all levels solvable, correct difficulty progression | Node.js extraction + test |
| 7. QA Testing | 40-point code-level checklist | Code analysis |
| 8. Register + Deploy | games-data.js, sitemap.html, sitemap.xml, git commit + push | Patch files + git |
| 9. Final Report | Structured summary of all phases | Output report |

## Current Status

Working on Phase 0-9 pipeline execution for puzzle games.

## Key Principles

1. **Browser Iron Law**: All browser tasks must use Kachilu Browser, not Hermes built-in browser
2. **Quality Gates**: Must pass 40-point QA checklist before deployment
3. **Incremental Development**: One feature at a time, verify before proceeding
4. **Visual Verification**: Screenshot confirmation required for UI/interaction correctness
5. **Data Consistency**: Verify 4-number sync (GAMES count, schema, HTML, category count) before completion

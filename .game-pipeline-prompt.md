## Key Principles

1. **Browser Iron Law**: All browser tasks must use Kachilu Browser, not Hermes built-in browser
2. **Quality Gates**: Must pass QA checklist before deployment
3. **Incremental Development**: One feature at a time, verify before proceeding
4. **Visual Verification**: Screenshot confirmation required for UI/interaction correctness
5. **Data Consistency**: Verify count sync (GAMES count, schema, HTML, about, manifest) before completion
6. **Saturation Awareness (2026-07-07)**: 目录已饱和(596款), 新游戏必须做差异化衍生, 优先选择 grep=0 的真零缺口
7. **Sibling Defusal**: Step 0 的 `kanban create --assignee` 会派发 sibling competitor; 必须 Step 0 后立即 archive
8. **Cron execute_code blocked**: cron profile 不允许 execute_code; 用 write_file/patch/terminal 替代
9. **rebuild_schema.js fixed**: 用 vm.runInContext 真实加载,不再用regex
10. **Partial-Registration Check**: 新游戏交付前必须 grep 检查是否已被 sibling 部分注册
11. **3-Method Verification**: 关卡验证应同时使用 (a) 独立 Python BFS, (b) 独立 Node BFS, (c) in-engine Node BFS (用 HTML 实际游戏规则)。三者都过 = 数学证明 + 引擎一致。
12. **Camoufox crash on rAF**: Camoufox headless 在 requestAnimationFrame 重渲染时可能崩溃。代码级 QA (3-method BFS + playtest simulation) 可替代浏览器交互测试。
13. **S级商业化品质标准 (2026-05-15)**: 每款游戏必须包含完整关卡系统 + 美术资源 + 音乐音效 + 100% QA
14. **RunningHub 美术**: API POST https://www.runninghub.cn/task/openapi/create, apiKey=c9daa99350384cfa90ce7eaacead3f7a
15. **MiniMax 音乐**: API POST https://api.minimax.io/v1/music_generation
16. **Reverse-move level gen**: 液体倾倒类游戏关卡用reverse-move generation最可靠
17. **Sukoro fixed-cell-set variant**: Sukoro原版(numbered/black pattern unknown)无法保证唯一解。固定numbered cell set后, 玩家只填values(1-4), 数值由邻居数决定 → 自动唯一。
18. **Pre-commit hook**: 会在commit时验证GAMES count vs HTML counts (header, footer, cat-count)。修改games-data.js后必须同步所有index.html中的count。
19. **Unicode emoji in games-data.js**: 用 Python surrogateescape 编码读写避免 UnicodeEncodeError (emoji surrogate pairs)

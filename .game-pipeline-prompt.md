# GameZipper Automated Game Publishing Pipeline Specification

## Phase 0-9 Execution Flow

### Phase 0: Market Research
- Search AppBrain/Poki/CrazyGames for gap categories
- Score candidates (≥18/25 points)
- Update `.game-pipeline-candidates.md` with top picks

### Phase 1: Candidate Selection
- Read `.game-pipeline-candidates.md`
- Pick top-scored game from NEXT_CANDIDATE queue
- Update status to IN_PROGRESS

### Phase 2: Competitive Benchmark
- Analyze top competitor (downloads, features, mechanics, monetization)
- Write BENCHMARK.md
- Identify unique selling points

### Phase 3: Game Development
- Build complete single-file HTML game
- Use `delegate_task(role='leaf', toolsets=['terminal','file'])` OR `write_file` directly
- Target: 27-30 levels, 6 difficulty tiers
- All levels must be solvable
- Include: Canvas rendering, touch events, Web Audio API, localStorage

### Phase 4: RunningHub Art
- Generate OG image/icon via RunningHub API
- Preferred: `runninghub_app.py` script with `--info` first
- Fallback: Direct curl to workflow ERNIE-Image-Turbo (ID: 2045418640316567553)
- Or fallback: Procedural CSS/SVG art

### Phase 5: Music Generation
- Primary: MiniMax API (`music-2.6-free`, `is_instrumental: true`)
- Fallback: Web Audio API procedural music
- Simple chord loop or scene-based BGM engine

### Phase 6: Level Verification
- For handcrafted levels: Python regex extraction
- For procedural generation: Node.js extraction + test
- Validate: all levels solvable, correct difficulty progression

## QA Execution Pattern (Cron Mode)

**Use grep + Node.js validation, NOT browser:**

```bash
cd /home/msdn/gamezipper.com

# Static checklist (grep-based)
grep -c "site-analytics.cap.1ktower.com" <slug>/index.html
grep -c "touch-action" <slug>/index.html
grep -c "application/ld+json" <slug>/index.html
grep -c "og:title" <slug>/index.html
grep -c "canonical" <slug>/index.html
grep "text-stroke" <slug>/index.html && echo "FORBIDDEN"

# JavaScript syntax validation (extract largest script block)
python3 -c "
import re
with open('<slug>/index.html') as f:
    html = f.read()
blocks = re.findall(r'<script>(.*?)</script>', html, re.DOTALL)
largest = max(blocks, key=len)
with open('/tmp/game_pure.js', 'w') as f:
    f.write(largest)
print(f'Extracted {len(largest)} chars')
"
node --check /tmp/game_pure.js

# Level validation (game-specific regex)
grep -c "cols:" <slug>/index.html  # Count levels
```

## Known Pitfalls

### Claude Code timeout (30min hard limit)
- Symptom: `timed out after 1800.0s`
- Check file existence and MD5 stability
- If file exists and passes validation → use it
- If not → fallback to `write_file` directly

### process(action='wait') clamped to 60s
- Use polling loop instead
- Check file progress each iteration
- Use MD5 stability check for completion

### Claude Code omits cleanup code
- Add `cleanup()` function with:
  - `cancelAnimationFrame`
  - `beforeunload` handler
  - `visibilitychange` handler
  - Audio context closure
  - Tracked `setInterval` cleanup

### Claude Code misses site-analytics script
- Always check in Phase 4 QA
- Patch in manually if missing

### QA checklist gaps
- Use `grep` for counting (not `read_file` which truncates)
- Skip JSON-LD blocks from JS validation
- Validate JSON-LD separately with Python `json.loads()`

### Phase 8 registration gaps
- Always load `gamezipper-new-game-checklist` skill
- Follow all 10 steps
- Run data consistency self-check at the end

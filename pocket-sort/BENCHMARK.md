## 1. Market Overview

## 2. Competitor Profiles

### 2.1 Pocket Sort: Coin Merge Puzzle (SayGames)

| Attribute | Details |
|---|---|
| **Developer** | SayGames Ltd |
| **Platform** | iOS & Android |

| **Package** | com.pocket.sort.coin.puzzle.game |
| **Launched** | March 2026 |

#### Core Mechanics
- **Sorting**: Tap to pick up coin stacks, place into slots, sorting by color.
- **Merging**: Identical coins combine into higher-value coins.
- **No timer**: Turn-based, untimed, relaxing.
- **Goal**: Sort all coins by color and merge to clear the board.

#### Special Coin Types
| Coin Type | Behavior |
|---|---|
| Standard | Rainbow of colors |
| **Frosted/Ice** | Immobilized until freed by matching adjacent coins |
| **Linked** | Affect adjacent pieces when merged (chain reactions) |
| **Hidden** | Reveal only after nearby moves |

#### Scoring and Progression
- No timer or countdown. Coins earned per level for boosters.
- Progressive difficulty: more colors, types, complex boards.

#### Power-ups
- Power-ups to clear tricky coins, reveal hidden pieces, unfreeze frosted coins.
- Available via gameplay and IAP.

#### Tutorial
- Animated hand guides. Simple tap-to-sort with immediate feedback.
- Gradual introduction of new coin types.

#### Visual Style
- **Bright, uncluttered 2D** with soothing color palette.
- Clean slot-based layout. Relaxing, no flashing lights.

#### Monetization
- Free-to-play with ads (interstitial/rewarded video) + IAP.
- 13 languages supported.

### 2.2 Magic Sort (Grand Games)

| Attribute | Details |
|---|---|
| **Developer** | Grand Games A.S. (Turkey) |
| **Platform** | iOS & Android |
| **Downloads** | 10,000,000+ |
| **Revenue** | $100K+/day IAP; 3.4M downloads/month |
| **Funding** | $30M (December 2024) |
| **Package** | com.grandgames.magicsort |
| **Launched** | June 26, 2024 |

#### Core Mechanics
- **Water Sort variant**: Sort colored water into test tubes until single color.
- **3 moves** per color typically needed per tube.
- Levels **randomly generated** for endless replayability.

#### Level Design
- **Normal, Hard, Super Hard** categories.
- Difficulty spikes every 10 levels (39, 49, 59).
- **1,038+ levels** (adding 100 per update).

#### Scoring
- **Stars** per level. **Magic Streak** with multipliers (up to x20).
- **Coins and diamonds** currencies.
- **Weekly Cup** with exclusive rewards and avatar frames.

#### Social Features
- **Teams system** with friends/strangers.
- **Leaderboards**: Individual, country-based, team.

#### Power-ups
- Special effects (e.g., Broom Streak). Magic Hat daily draws.
- Running out of moves costs gold, incentivizing booster use.

#### Tutorial
- Smooth FTUE, one-finger controls.
- Interstitial ads delayed until after level 24.

#### Visual Style
- **Magical/potion theme** with test tubes.
- Semi-masked visual cues for unpredictability.

#### Monetization
- **Hybrid**: IAP + Ads. Interstitials only after level 24.
- Rewarded videos for lives/boosters. Diverse IAP bundles.
- Ad revenue primary; IAP secondary.

## 5. Key Takeaways for Development

### What to Emulate

| Strategy | Source | Implementation |
|---|---|---|
| **Merge+Sort dual mechanic** | Pocket Sort, iColorCoin | Sort by color AND merge by value |
| **Special item variety** | Pocket Sort | Ice, linked, hidden coins add depth |
| **No timer, pure logic** | All competitors | Untimed = table stakes |
| **Hand-guided tutorial** | Hexa Sort, Pocket Sort | Non-intrusive FTUE |
| **Delayed interstitial ads** | Magic Sort (after lvl 24) | Protect early retention |
| **Streak/reward systems** | Magic Sort | Multipliers on consecutive wins |
| **Progress bar per level** | Hexa Sort | Visual progress toward completion |
| **Satisfying merge animations** | Hexa Sort | Animations + sound = retention |

### What to Avoid

| Pitfall | Source | Lesson |
|---|---|---|
| **Ad overload** | Water Sort | Ad hell kills retention |
| **No meta-progression** | Water Sort | Pure gameplay has low LTV |
| **No social features** | Most sort games | Teams/leaderboards boost engagement |
| **Aggressive IAP** | Goods Sort | Users revolt against pay-walls |

### Recommended Feature Priorities

#### Tier 1 - Must Have (Launch)
1. Sort-by-color + merge-by-value dual mechanic
2. No timer, turn-based gameplay
3. Special coin types (ice, linked, hidden)
4. Animated hand-guide tutorial
5. Level-based progression with gradual difficulty
6. Rewarded video ads for boosters (no forced interstitials early)
7. Undo, Shuffle, and hint power-ups
8. Progress bar per level

#### Tier 2 - Should Have (Post-Launch)
1. 3-star rating per level
2. Coin economy with shop
3. Daily challenges / login calendar
4. Streak multiplier system
5. Cosmetic coin/slot skins
6. 1,000+ levels or procedural generation
7. Multiple visual themes / world progression

#### Tier 3 - Nice to Have (Live Ops)
1. Teams / social features
2. Weekly competitive events / leaderboards
3. Limited-time events with exclusive rewards
4. Piggy Bank savings mechanic
5. Safety Net / continue-on-failure
6. Seasonal themes and coin designs

### Monetization Strategy

- **~65% Ad Revenue**: Rewarded videos (voluntary) + controlled interstitials (after transitions)
- **~35% IAP Revenue**: Coin packs (\/usr/bin/bash.99-\.99), Piggy Bank (\.99), Starter Bundles, Ad Removal (\.99-\.99)
- **Key principle**: Ads should feel optional and rewarding, not punitive.

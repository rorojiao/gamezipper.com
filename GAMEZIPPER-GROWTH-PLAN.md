## 🚨 发现的关键问题（按严重程度排序）

### P0 - 严重阻塞
1. **6款游戏在首页无链接（孤立页面）**：arrow-escape, bubble-pop, fruit-slash, pixel-logic, tile-dynasty, t-rex
2. **sushi-stack/babylon.js 7.6MB** — 单文件拖垮整站性能
3. **游戏间交叉链接极弱** — 大部分游戏仅链接 1-2 个其他游戏
4. **BI tracker 未上线** — 无法获取 UV/设备/地区等精细数据
5. **Monetag API 不可用** — 无法获取广告收入数据

### P1 - 高优先级
6. **零 WebP 图片** — 269 张 PNG/JPG 全部未优化
7. **5 个新游戏缺 OG Image** — 社交分享无图
8. **TTFB 0.5-0.8s** — 偏高（Vercel 应该更快）
9. **DuckDuckGo 仅收录 ~10 条** — Google/Bing 收录数不明但可能偏低
10. **2 个新游戏缺 BreadcrumbList schema**

### P2 - 中优先级
11. **博客 SEO 优化空间大** — 239 篇博客但 SEO 元素未全面检查
12. **缺少 Bing Webmaster Tools 验证**
13. **无外链建设** — 0 主动 backlink 获取
14. **无社交媒体运营** — TikTok/YouTube/Pinterest 零存在
15. **无游戏评分系统**

## 🎯 优化方案（分阶段执行）

### 第一阶段：止血修复（本周完成）

#### 1.1 修复孤立页面
- [ ] 确认首页 games-data.js 中 44 款游戏全部有链接
- [ ] 修复 6 个孤立页面的首页入口

#### 1.2 性能紧急优化
- [ ] sushi-stack/babylon.js 替换为 CDN 模块化加载或轻量引擎
- [ ] TTFB 优化：检查 Vercel 缓存策略、启用 ISR

#### 1.3 内链结构重建
- [ ] 每个游戏页推荐至少 4-6 个同类游戏
- [ ] 建立分类页→游戏页→相关游戏的三级内链网

#### 1.4 SEO 元素补全
- [ ] 5 个新游戏补 OG Image
- [ ] 2 个新游戏补 BreadcrumbList schema
- [ ] 所有页面 schema 完整性检查

#### 1.5 BI 系统
- [ ] 将 bi.gamezipper.com/t.js 嵌入生产页面
- [ ] 验证事件上报正常

### 第二阶段：性能与收录提升（1-2周）

#### 2.1 图片优化
- [ ] 全站 269 张图片 PNG/JPG → WebP 批量转换
- [ ] 实施懒加载

#### 2.2 收录加速
- [ ] 注册 Bing Webmaster Tools + 提交 sitemap
- [ ] 验证 IndexNow ping 机制正常工作
- [ ] 手动提交新游戏 URL 到 Google Search Console
- [ ] 检查 Google 实际收录数，诊断未收录原因

#### 2.3 Core Web Vitals
- [ ] 首页 LCP/FID/CLS 优化目标：<2.5s / <100ms / <0.1
- [ ] 所有游戏页性能审计

### 第三阶段：内容增长（2-4周）

- [ ] 审查现有 239 篇博客的 SEO 质量

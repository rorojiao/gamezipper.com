# Monkey Mart 运行时数值提取报告

## 任务目标
通过运行时方法提取Monkey Mart游戏数值（商品价格、升级成本、顾客频率等）。

## 执行过程

### 1. 浏览器环境访问
- 使用Kachilu Browser访问 `https://poki.com/en/g/monkey-mart`
- 游戏通过Poki平台托管，加载于 `games.poki.com` 的iframe中

### 2. 游戏引擎确认
控制台输出确认：
```
Made with Defold
Registered utf8 Extension
Registered astar Extension
```
确认游戏使用 **Defold 引擎**开发。

### 3. 跨域访问限制分析

#### 尝试的方法：
1. **直接iframe.contentWindow访问**
   - 结果：失败
   - 错误：`Blocked a frame with origin "https://poki.com" from accessing a cross-origin frame.`
   - 原因：Poki使用跨域隔离，games.poki.com与poki.com非同源

2. **postMessage通信尝试**
   - 发送消息给游戏iframe
   - 结果：游戏未响应任何postMessage请求
   - 说明游戏不监听来自父窗口的调试消息

3. **init_script注入**
   - Kachilu不支持`inject_init_script`命令

4. **CDP (Chrome DevTools Protocol) 连接**
   - 已建立CDP连接 (ws://127.0.0.1:32943)
   - 但Kachilu的eval命令无法直接在不同frame上下文中执行脚本
   - 原因：frame上下文隔离，eval只能在当前主frame执行

### 4. 限制原因总结

**WASM/Defold游戏保护机制：**

| 保护层级 | 机制 | 绕过难度 |
|---------|------|---------|
| 跨域隔离 | Poki主站与游戏iframe非同源 | 无法绕过 |
| 运行时隔离 | 游戏代码运行在隔离的JavaScript上下文 | 无法绕过 |
| postMessage过滤 | 游戏未实现消息监听接口 | 无法绕过 |
| 内存保护 | 游戏数据存储在私有内存空间 | 无法绕过 |

## 无法提取的数据

由于上述限制，以下数据无法通过运行时方法提取：
- 所有商品价格表
- 升级成本表
- 顾客生成频率参数
- 员工配置数据
- 各集市解锁条件
- 生产速度/冷却时间

## 建议

若需要获取游戏数值，建议采用以下方法：

1. **静态逆向分析**
   - 下载游戏WASM二进制文件
   - 使用WASM分析工具（如IDA、Ghidra + WASM插件）
   - 搜索游戏数值所在的内存段

2. **网络抓包分析**
   - 捕获游戏与服务器的通信数据
   - 分析可能的配置API或存档数据

3. **直接联系开发者**
   - TinyDobbins (开发者) 可能愿意分享游戏数据

## Core Mechanics Analysis

### Primary Loop
1. **Target Display**: Show target sum (e.g., "23")
2. **Swipe Input**: Drag finger through numbers to select sequence
3. **Auto-Calculation**: Real-time running sum displayed
4. **Success/Fail**: If sum matches target → next level; else → retry

### Input Handling
- **Swipe Start**: Touch first tile (animates selection)
- **Drag to Add**: Pass over tiles to include in sequence
- **Backtrack**: Swipe back to deselect last tile
- **Swipe End**: Lift finger to submit (auto-check sum)
- **Undo**: Button to clear selection and retry

### Difficulty Scaling (5 Tiers)
| Tier | Grid Size | Target Range | Max Moves | Time Limit |
|------|-----------|--------------|-----------|------------|
| Tutorial | 4x4 | 5-15 | ∞ | None |
| Easy | 4x4 | 10-25 | 5 | 30s |
| Medium | 5x5 | 20-40 | 4 | 25s |
| Hard | 5x5 | 30-60 | 3 | 20s |
| Expert | 6x6 | 40-100 | 2 | 15s |

## Browser Implementation Strategy

### What We'll Deliver
1. **Mobile-First Touch**: Swipe gestures (touchstart/move/end)
2. **Mouse Fallback**: Drag-select for desktop
3. **Visual Feedback**: 
   - Selected tiles: neon glow + scale animation
   - Running sum: HUD display with diff color (green if match, red if over)
   - Success: particle burst + confetti
   - Fail: shake animation + retry message
4. **30 Levels**: 5 tiers × 6 levels (tutorial → expert progression)
5. **Progress Saving**: LocalStorage (level + stars)

### Technical Specs
- **File Size**: < 50KB (gzipped) HTML5 single file
- **Canvas Size**: Responsive, max 600px width
- **Font**: Google Fonts (Inter or similar for numbers)
- **Colors**: Dark theme (#0f0f0f bg), neon accents (#00ff88, #ff00ff)
- **Performance**: 60fps on mid-range mobile

## Uniqueness vs Catalog

### Catalog Gaps Confirmed
- ❌ **No number-swipe games**: Catalog has puzzle (382 games) but zero with swipe-sum mechanics
- ❌ **No real-time math**: Number Match / Math 24 use static selection
- ❌ **No gesture-heavy puzzle**: Most games are tap-click, not swipe-heavy

### Competitive Advantages
1. **Zero browser competition**: First-to-market on web
2. **Mobile-native feel**: Swipe gestures feel like native app
3. **Satisfying feedback**: Particle effects + haptic vibration support
4. **Sticky progression**: 30 levels with star ratings encourage replay

## Risk Mitigation

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| **Swipe misfires** | High | Medium | Touch tolerance zone + visual snap feedback |
| **Math anxiety** | Medium | Medium | Tutorial emphasizes addition only (no subtraction) |
| **Browser performance** | Low | High | Keep tile count ≤ 36 (6×6 grid), use requestAnimationFrame |

## Decision Rationale

### Why Sum Swipe?
1. **Market validation**: 120M+ mobile downloads = proven demand
2. **Low execution complexity**: Grid + swipe logic is well-understood
3. **High uniqueness score**: 0/467 catalog overlap
4. **Monetization-friendly**: Fast sessions (1-2 min) = high ad rotation

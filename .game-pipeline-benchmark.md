## Core Mechanics Analysis

### Primary Loop
1. **Grid Display**: Show a grid of N×N items, all identical except ONE
2. **Identify**: Player finds the one different item
3. **Tap**: Tap the odd item to score
4. **Advance**: Next level (larger grid, subtler difference, time pressure)
5. **Streak/Miss**: Correct = points + streak; Wrong = lose time/life

### Difficulty Scaling (5 Tiers)
| Tier | Grid Size | Difference Type | Time Limit | Items |
|------|-----------|-----------------|------------|-------|
| Tutorial | 3×3 | Color (obvious) | None | 9 |
| Easy | 4×4 | Color/Shape | 30s | 16 |
| Medium | 5×5 | Shape/Pattern | 20s | 25 |
| Hard | 6×6 | Pattern/Hue | 15s | 36 |
| Expert | 7×7 | Subtle Hue/Letter | 10s | 49 |

### Visual Themes (rotating)
1. **Colors**: Identical-colored circles, one slightly different hue
2. **Emojis**: Grid of same emoji, one slightly different variant
3. **Shapes**: Same shape, one rotated/modified
4. **Numbers**: Same number, one different
5. **Letters**: Same letter, one different case/variant

## Browser Implementation Strategy

### What We'll Deliver
1. **30 Levels**: 6 per tier × 5 tiers (tutorial → expert)
2. **Mobile-First Tap**: Touch/click the odd item
3. **Visual Feedback**:
   - Correct: green pulse + particle burst + combo counter
   - Wrong: red shake + time deduction
   - Streak indicator with multiplier
4. **Multiple Themes**: Rotating visual types keep gameplay fresh
5. **Progress Saving**: LocalStorage (level + stars + best streak)

### Technical Specs
- **File Size**: < 40KB (gzipped) HTML5 single file
- **Canvas Size**: Responsive, max 500px width
- **Performance**: 60fps, instant tap response
- **Colors**: Modern dark theme (#1a1a2e bg), neon accents
- **Font**: System sans-serif for maximum performance

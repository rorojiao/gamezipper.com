### Selected Candidate: **Tetravex** (slug: `tetravex`)

**Mechanic**: Edge-matching tile puzzle. Each square tile has a number/color on
each of its 4 edges. Place tiles in a grid so that adjacent edges match. Classic
GNOME / gcompris game with a 30-year pedigree. Pure logic, no physics.

**Gap Score (25-pt rubric)**:
- **Market gap (≥5M downloads, no browser clone)**: 9/10 — Tetravex ships in
  GNOME Games (installed on every Ubuntu/Fedora desktop, ~50M+ Linux users),
  gcompris (10M+ educational downloads), and Microsoft Entertainment Pack
  ("Tetravex" was a Win 3.1 staple). Zero quality browser clone exists; the
  GNOME version requires Linux + GNOME desktop. Web gap is real and large.
- **Search volume**: 4/5 — "tetravex" has steady niche search volume (Ubuntu
  manpage ranks), recognized name in puzzle community, recurring requests for
  a web version on puzzle forums.
- **Monetization potential**: 4/5 — Casual logic puzzle, perfect for
  interstitials every 5 levels + banner. Long sessions (logic thinkers play
  20+ min). High replay via daily challenge + tier system.
- **Technical feasibility**: 3/3 — Pure DOM/Canvas, no physics engine. Tile
  drag-drop + adjacency check is trivial. Level generation is a constructive
  solver problem (well-understood, CSP-style).
- **Uniqueness**: 2/2 — Mechanic is distinct from every catalog entry:
  - vs Jigsaw (shape-match, not edge-number-match)
  - vs Slitherlink (draw-loop, not place-tile)
  - vs Dominosa (partition grid into domino pairs, not edge-match squares)
  - vs Nikoli family (no existing edge-matching tile-placement game)

**Total: 22/25** ✅ EXCEEDS 18/25 threshold

### Why Tetravex over the R27 leftovers
1. R27's two unshipped leftovers are both occupied (Domino Toppler) or
   mechanic-cannibalizing (Conveyor Factory vs just-shipped conveyor-sort).
2. Tetravex is a NAMED, RECOGNIZED puzzle (Ubuntu manpage, GNOME Games suite,
   Microsoft Entertainment Pack lineage) — name recognition drives search.
3. Mechanic is 100% distinct from the 525-game catalog (verified via the
   R56/R58 mandatory recipe: slug grep 0, display-name grep 0, per-word grep 0,
   dir absent).
4. Constructive-generation level design is well-trodden territory for this
   pipeline (the CSP solver/generator pattern from dominosa, futoshiki, etc.
   applies directly).
